# Learning Content Style Guide

이 문서는 한국어·영어 학습 원고, 시뮬레이션 문구, 퀴즈를 작성할 때 지키는 기준이다. 목표는 설명을 사람 교사가 다듬은 교재처럼 구체적이고 담백하게 유지하는 것이다.

## 기본 문체

- 한국어는 `이다·한다`식 평서문을 사용한다.
- 영어는 짧고 직접적인 문장을 사용한다.
- 영어 문장의 어순이나 추상적 동사를 그대로 옮기지 않는다. 한국어 설명은 따로
  읽어 보고, "아이디어를 반복해서 사용한다"보다 실제로 무엇이 표시되고 어떤
  확률이 변하는지를 구체적으로 쓴다.
- 기능이나 자료를 만들었다는 개발자 관점보다 학생이 보게 될 현상과 이유를 쓴다.
- “고등학생도 이해하기 쉽게 골랐다”, “좋은 자료를 연결했다”처럼 콘텐츠 제작 과정을 설명하지 않는다.
- “쉽게 말하면”, “핵심은 간단하다”, “이것이 바로”, “In short”, “The main idea is” 같은 상투적 연결어로 결론을 대신하지 않는다.

## 강조와 용어

- 설명 카드 본문에서 장식용 `**굵게**` 표기를 사용하지 않는다. 카드 제목, 수식, 용어표가 이미 구조를 제공한다.
- 용어는 처음 등장할 때 평문으로 뜻을 설명하고 용어표에 같은 정의를 둔다.
- control, target, big-endian처럼 표기 방식이 여러 개인 개념은 이 앱의 기준을 명시한다.
- CNOT은 `CNOT(control, target)` 순서로 쓰고, 회로에서는 `●`가 control, `⊕`가 target이라고 설명한다.
- 두 큐비트 게이트의 공통 표기와 CNOT의 기저 상태 동작은 Level 5에서 처음 소개하고, Level 6부터는 그 정의를 반복하지 않고 얽힘 생성에 연결한다.

## 설명의 흐름

하나의 예시나 실험은 다음 순서를 갖는다.

1. 입력 또는 시작 상태를 밝힌다.
2. 어떤 조작이 어느 대상에 적용되는지 쓴다.
3. 계산이나 관찰 결과를 제시한다.
4. 학습자가 최종적으로 무엇을 얻었는지 한 문장으로 닫는다.
5. 그 결과가 개념상 무엇을 뜻하는지 설명한다.

“보정을 적용한다”에서 끝내지 않고 “그래서 Bob이 Alice가 보내려던 상태를 얻는다”까지 써야 한다. 표나 그래프 뒤에도 어떤 열이나 변화가 질문의 답인지 명시한다.

## 수식과 예시

- 결과식이 갑자기 나오지 않도록 필요한 중간 상태를 보여 준다.
- 항의 순서는 앞에서 정의한 식과 대응되게 유지한다. 예를 들어 $\lvert\psi\rangle=\alpha\lvert0\rangle+\beta\lvert1\rangle$이면 $X\lvert\psi\rangle=\alpha\lvert1\rangle+\beta\lvert0\rangle$처럼 $\alpha$ 항을 먼저 쓴다.
- 한 큐비트의 일반적인 순수 상태를 입력받을 때는 전체 위상을 제거한 $\lvert\psi\rangle=\lvert\alpha\rvert\lvert0\rangle+e^{i\phi}\lvert\beta\rvert\lvert1\rangle$ 표기를 기본으로 사용한다. 실수 진폭만 쓰는 예시는 $\phi=0$ 또는 $\pi$인 특수한 경우임을 밝힌다.
- 수식을 건너뛰어도 되는 경우에는 “어려우면 넘어가도 된다”만 반복하지 말고, 그 식이 말하는 물리적 의미를 바로 덧붙인다.
- 추상 설명 뒤에는 구체적인 상태, 측정값, 회로 또는 생활 예시를 하나 이상 둔다.
- 같은 그림을 반복해서 넣기보다 현재 설명의 질문을 직접 보여 주는 그림을 사용한다.

## 시뮬레이션 문구

- 실행 전에는 무엇을 바꾸고 무엇을 관찰할지 설명한다.
- 실행 후에는 값만 표시하지 않고 그 값이 실험 질문에 답하는 방식을 적는다.
- 서로 종속된 입력을 각각 조절하게 하지 않는다. 정규화처럼 한 값이 다른 값을 정하면 나머지는 자동 계산한다.
- 내부 구현, 배포 방식, 임시 대체 자료에 관한 문구는 학습 화면에 노출하지 않는다.

## 한영 동기화

- 한국어와 영어 원고는 카드 수, 그림, 수식, 예시, 최종 결론을 맞춘다.
- 번역할 때 문장 구조를 억지로 같게 만들 필요는 없지만 정보는 빠뜨리지 않는다.
- 새 용어와 UI 키는 두 언어의 구조화 데이터에 함께 추가한다.

## 수정 전 점검

- 결과만 제시하고 중간 이유를 생략하지 않았는가?
- 예시의 마지막에 학습자가 얻은 결과가 적혀 있는가?
- 굵은 글씨나 상투적 요약이 문장 내용을 대신하지 않는가?
- 표기 순서와 control/target이 앱 기준과 일치하는가?
- 한국어와 영어에 같은 수식·그림·결론이 있는가?
