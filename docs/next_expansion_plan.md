# Quantum Lab 후속 확장 계획

이 문서는 Level 12까지 완성된 현재 학습 흐름과 이후 확장 순서를 정리한다. 2026-07-20에는 어려운 Deutsch 알고리즘을 간섭 심화 뒤의 Level 11로 옮기고, 노이즈·회로 읽기·측정 통계·간섭 심화를 Level 7-10으로 재배치했다.

고등학생 대상에서는 Grover search, QFT, 변분 알고리즘을 바로 넣기보다 Level 0-12의 내용을 더 깊고 친절하게 다지는 편이 낫다. 고급 알고리즘은 "나중에 이런 세계가 열린다" 정도로 자료실이나 특별 주제로 미뤄 둔다.

## 기본 방향

- PC/태블릿 최적화, 모바일 호환을 유지한다.
- 새 레벨은 설명, 시뮬레이션, 자료실, 퀴즈, 한국어/영어 데이터를 함께 추가한다.
- 어려운 수식은 필수가 아니라 의미를 돕는 도구로 둔다.
- 새 레벨은 기존 Level 0-12를 복습하고 연결하는 방식으로 설계한다.
- Qiskit/Cirq는 당장 필수 의존성으로 넣지 않는다. 먼저 앱 안의 작은 시뮬레이터와 외부 실습 링크로 연결한다.

## Qiskit/Cirq 적용 원칙

Qiskit과 Cirq는 실제 양자 회로를 작성하고 실행하는 Python 기반 도구다. 그러나 초중급 학습앱에 바로 필수 의존성으로 넣으면 설치 시간, 배포 실패, 버전 변화가 학습 목표보다 커질 수 있다.

권장 방식:

1. Level 8-12는 앱 자체 시뮬레이터로 개념을 먼저 다룬다.
2. 각 레벨 자료실에 IBM Quantum Composer, Qiskit Learning, Cirq 튜토리얼, Colab 링크를 붙인다.
3. SDK 실습은 나중에 `SDK Lab` 또는 별도 문서로 분리한다.
4. 기본 `requirements.txt`에는 Qiskit/Cirq를 넣지 않고, 필요해지면 `requirements-sdk.txt`를 별도로 둔다.

참고 공식 문서:

- Qiskit 설치: https://quantum.cloud.ibm.com/docs/en/guides/install-qiskit
- Cirq 저장소 및 설치 안내: https://github.com/quantumlib/Cirq

## 현재 학습앱 심화 흐름

### Level 8 · 회로를 읽는 법

핵심 질문: 지금까지 배운 게이트와 측정을 하나의 회로 그림으로 어떻게 읽는가?

Level 3, 5, 6을 다시 묶는 레벨이다. 새 알고리즘보다 "회로 문법"을 익히는 데 집중한다.

학습 내용:

- 큐비트 선, 게이트, 제어점, 측정 기호
- 회로는 왼쪽에서 오른쪽으로 읽는다는 규칙
- 같은 회로라도 측정을 어디에서 하느냐에 따라 해석이 달라진다는 점
- Bell 회로를 통해 `H`, `CNOT`, 측정의 연결 복습

시뮬레이션:

- 미리 준비된 회로 카드 선택: `X`, `H`, `H + Measure`, `H + CNOT`, `Bell + Measure`
- 회로 그림, 상태 설명, 측정 결과 막대그래프를 함께 표시
- Qiskit/Cirq 코드는 읽기 전용 예시로만 제공

### Level 9 · 측정과 통계 다시 보기

핵심 질문: 한 번의 측정 결과와 많은 반복 측정 결과는 왜 다르게 해석해야 하는가?

Level 0, 1, 2, 7을 심화한다. 고등학생에게 가장 중요한 오해인 "한 번 봤으니 상태를 안다"를 정리한다.

학습 내용:

- 한 번 측정과 반복 측정의 차이
- 확률 분포, 표본 수, 흔들림
- 노이즈가 있는 결과와 원래 양자 확률의 구분
- 실험 결과를 읽을 때 평균만 보지 않고 퍼짐도 함께 보는 이유

시뮬레이션:

- 같은 상태를 10회, 100회, 1000회 측정하며 히스토그램 비교
- 노이즈 없음/약한 노이즈/강한 노이즈 전환
- 결과 표에 영어 그래프 라벨 사용

### Level 10 · 간섭을 더 깊게 보기

핵심 질문: 가능성이 더해질 때 왜 어떤 결과는 커지고 어떤 결과는 사라지는가?

Level 4를 심화하고 다음 Level 11의 Deutsch 알고리즘으로 연결한다. 이 단계에서는 알고리즘 자체보다 간섭의 의미를 더 친절하게 설명한다.

학습 내용:

- 진폭의 부호와 위상
- 보강 간섭과 상쇄 간섭
- 양자 알고리즘에서 쓸모없는 답을 줄이고 원하는 패턴을 키우는 방식
- 소리, 물결, 빛의 간섭 예시와 양자 간섭의 차이

시뮬레이션:

- 두 경로의 위상 차이를 바꾸면 밝은 결과/어두운 결과가 바뀌는 화면
- `+`와 `-` 진폭이 만나 사라지는 간단한 표
- 수식 없이 먼저 보고, 원하는 학생만 수식 카드 열기

### Level 11 · 간단한 양자 알고리즘

핵심 질문: 오라클이 남긴 위상 관계를 간섭으로 어떻게 읽는가?

Level 10에서 위상 차이와 간섭을 충분히 본 뒤 Deutsch 알고리즘으로 연결한다. 오라클 호출 횟수보다 `(-1)^{f(x)}` 위상 표식과 마지막 H의 역할을 중심으로 설명하며, 상세 유도식은 선택형 접기 영역에 둔다.

시뮬레이션:

- 네 가지 오라클의 진리표와 constant/balanced 분류 비교
- 오라클 뒤 위상 표식과 마지막 H 뒤 측정 결과 연결
- 회로 단계와 상세 상태표는 필요할 때만 펼쳐 보기

### Level 12 · 얽힘과 정보의 한계

핵심 질문: 얽힘은 강한 관계를 만들지만, 왜 그것만으로 빛보다 빠른 통신은 안 되는가?

Level 6을 심화한다. 얽힘을 신비한 순간이동처럼 오해하지 않게 정리하는 레벨이다.

학습 내용:

- 상관관계와 인과관계의 차이
- 같은 기준으로 측정할 때 드러나는 강한 관계
- 한쪽 결과만 보면 여전히 랜덤이라는 점
- 양자 순간이동과 고전 통신의 필요성은 맛보기로만 소개

시뮬레이션:

- 두 사람이 같은 기준/다른 기준으로 측정하는 결과 비교
- 한쪽 데이터만 볼 때와 두 데이터를 맞춰 볼 때의 차이
- Bell 실험은 정성적 직관 중심으로 소개

### Level 13 · 오류와 보정의 현실

핵심 질문: 실제 양자컴퓨터는 왜 오류를 피할 수 없고, 어떻게 줄이려 하는가?

Level 7을 심화한다. 완전한 양자 오류 정정보다 "오류를 발견하고 줄인다"는 현실감을 먼저 준다.

학습 내용:

- 비트 뒤집힘, 위상 흐트러짐, 측정 오류
- 오류 완화와 오류 정정의 차이
- 반복 부호의 고전 버전
- 큐비트를 단순 복사할 수 없어서 양자 오류 정정이 어려운 이유

시뮬레이션:

- 1비트 전송과 3비트 반복 전송 비교
- 노이즈가 커질수록 다수결도 한계가 있음을 표시
- 양자 버전은 원리 카드로 간단히 연결

### Level 14 · 실제 양자컴퓨터 사용 맛보기

핵심 질문: 지금까지 배운 회로를 실제 도구에서는 어떻게 표현하는가?

여기서 Qiskit/Cirq를 본격 실습으로 연결한다. 단, 앱 배포 의존성은 그대로 가볍게 유지한다.

학습 내용:

- IBM Quantum Composer로 Bell 회로 만들기
- Qiskit/Cirq 코드가 회로 그림과 어떻게 대응되는지 보기
- 로컬 설치 대신 Colab 또는 공식 웹 도구를 먼저 사용

시뮬레이션:

- 앱 안에서는 회로와 코드 예시를 나란히 보여준다.
- 실제 실행은 외부 링크 또는 별도 `SDK Lab` 문서로 연결한다.

## 고급 트랙으로 미룰 주제

다음 주제는 가치가 크지만, 현재 Level 11 바로 다음에 넣기에는 추상도가 높다. 기본 개념을 충분히 다진 뒤 특별 레벨이나 심화 코스로 분리한다.

- Grover search: 진폭 증폭을 이해한 뒤 다룬다.
- QFT와 위상 추정: 복소수 위상과 주기 찾기 직관이 더 쌓인 뒤 다룬다.
- 변분 알고리즘, VQE, QAOA: 최적화와 에너지 함수 직관이 필요하므로 뒤로 미룬다.
- Shor 알고리즘: 공개키 암호와 주기 찾기 배경이 필요하므로 고급 자료실 주제로 둔다.

## Photon Heist Chapter 3 이후

### Chapter 3 · Focus Job

학습 내용: 볼록렌즈, 오목렌즈, 초점, 상의 형성

게임 아이디어:

- 여러 광선을 한 점에 모아 센서를 켠다.
- 볼록렌즈는 모으고, 오목렌즈는 퍼뜨린다.
- 목표는 한 줄 레이저가 아니라 광선 묶음의 모양이다.

### Chapter 4 · Spectrum Vault

학습 내용: 프리즘, 파장, 색 분산

게임 아이디어:

- 흰빛을 색으로 나누고, 특정 색만 목표 센서에 넣는다.
- 빨강/파랑의 굴절 차이를 퍼즐 규칙으로 사용한다.
- 색이 섞이면 실패하는 가짜 센서를 둔다.

### Chapter 5 · Polarization Cipher

학습 내용: 편광, 필터 각도, 측정 기준

게임 아이디어:

- 필터 각도를 돌려 통과 확률을 조절한다.
- 서로 다른 기준에서 측정하면 결과가 달라진다는 점을 게임 규칙으로 만든다.
- BB84로 넘어가기 전 준비 단계로 쓴다.

### Chapter 6 · Wave Protocol

학습 내용: 간섭, 위상, 회절

게임 아이디어:

- 두 경로의 빛을 다시 합쳐 밝은 곳과 어두운 곳을 만든다.
- 경로 길이 차이로 위상을 조절한다.
- 목표 센서는 밝게, 보안 센서는 어둡게 만드는 퍼즐로 구성한다.

### Chapter 7 · Photon Operations

학습 내용: 단일 광자, 확률, 반복 측정

게임 아이디어:

- 한 번의 검출은 랜덤하지만, 여러 번 반복하면 확률 패턴이 드러난다.
- 빔스플리터와 검출기를 사용한다.
- 학습앱 Level 0-2와 직접 연결한다.

### Chapter 8 · Quantum Heist

학습 내용: 큐비트, 중첩, 측정, BB84

게임 아이디어:

- 앞 챕터의 편광/간섭 장치를 양자정보 장치로 다시 해석한다.
- 올바른 측정 기준을 고르면 신호를 읽고, 틀린 기준은 오류를 만든다.
- 마지막 챕터는 "빛의 도둑"에서 "정보의 도둑을 잡는 보안"으로 반전시킨다.

## 구현 순서 제안

1. Level 8-12의 회로 읽기, 통계, 간섭, Deutsch 알고리즘, 얽힘 한계 흐름은 구현 완료 상태로 유지·보완한다.
2. Level 13에서 오류와 보정의 현실을 다룬다.
3. Level 14에서 Qiskit/Cirq를 선택형 실습으로 연결한다.
4. Grover/QFT/변분 알고리즘은 고급 트랙으로 따로 계획한다.
5. Photon Heist는 Chapter 3 렌즈 챕터부터 순서대로 확장한다.
