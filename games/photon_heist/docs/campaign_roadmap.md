# Photon Heist 개발 로드맵

## 최종 방향

플레이어가 거울과 렌즈 같은 고전 광학 장치를 먼저 직접 조작한 뒤,
동일한 장치를 파동·광자·큐비트 관점에서 다시 해석하도록 구성합니다.
최종 목표는 공식을 암기하는 게임이 아니라, 실험을 설계하고 결과를
예측하는 양자정보 학습 게임입니다.

기존 6개 프로토타입 계획은 아래 8개 챕터로 확장합니다. 렌즈와 프리즘은
유지하고, 양자정보로 연결하는 데 필요한 편광과 측정 단계를 독립시켰습니다.

| 순서 | 챕터 | 핵심 개념 | 예상 스테이지 | 상태 |
| --- | --- | --- | ---: | --- |
| 1 | Mirror Mission | 직진, 반사 | 10 | ✅ 구현 완료 |
| 2 | Glass Route | 굴절, 굴절률, 전반사, 반사 결합 | 10 | 🔧 게임성 검증 중 |
| 3 | Focus Job | 볼록·오목 렌즈, 초점 | 8 | 설계 예정 |
| 4 | Spectrum Vault | 프리즘, 파장, 색 분리 | 8 | 설계 예정 |
| 5 | Polarization Cipher | 편광, 필터 각도, 측정 기저 | 8 | 설계 예정 |
| 6 | Wave Protocol | 간섭, 위상, 회절 | 10 | 설계 예정 |
| 7 | Photon Operations | 광자, 검출, 확률, 반복 측정 | 10 | 설계 예정 |
| 8 | Quantum Heist | 큐비트, 중첩, 측정, BB84 | 10 | 장기 목표 |

전체 목표 분량은 약 74개 스테이지입니다. 처음부터 모두 만들지 않고,
각 챕터의 핵심 4개 스테이지를 먼저 검증한 뒤 8~10개로 확장합니다.

## Prototype 1. Mirror Mission — 거울과 반사

### 학습 목표

- 빛의 직진과 반사를 경로로 예측한다.
- 45°와 135° 거울을 조합한다.
- 실제 경로에 영향을 주는 장치와 미끼를 구분한다.

### 게임 장치

- 레이저, 회전 거울, 벽, 목표 센서, 가짜 센서
- 별점, 추가 도전, 미끼 거울, 스테이지 지도

### 상태

- 10개 스테이지 구현 및 전수 풀이 검증 완료
- Stage 2-2 수납 거울 선택과 계획 → 발사 → 결과 세로 단면, 3발 제한, 실패 경로 잔상 구현
- Stage 2-3 같은 행·열을 따라 후보가 다시 분기되는 다단계 거울 네트워크 구현
- Stage 3-1 보안 키 선획득 후 거울망을 재구성해 탈출하는 순차 2발 작전 구현
- Stage 3-2 노란 키 → 청록 키 → 최종 센서를 중앙 거울의 세 상태로 여는 순차 3발 작전 구현
- 이후 챕터에서도 거울과 보안 센서를 공통 장치로 재사용

## Prototype 2. Glass Route — 유리 블록과 굴절

### 학습 목표

- 경계면의 법선과 입사각·굴절각을 구분한다.
- 공기에서 유리로 갈 때 빛이 법선 쪽으로 꺾임을 예측한다.
- 유리에서 공기로 나올 때 법선에서 멀어짐을 예측한다.
- 평행 유리판의 출사광은 입사광과 평행하지만 옆으로 이동함을 이해한다.
- 임계각을 넘으면 전반사가 일어남을 체험한다.

### 새 게임 장치

- 유리 영역과 공기–유리 경계
- 고정 유리판과 회전 가능한 레이저
- 굴절률 표시기와 법선 보조선
- 계획·발사 버튼, 발사 잔상, 정보 조각, 고정 거울

### 10개 스테이지 구성 (콘텐츠 구현, 게임성 재검증 중)

| ID | 제목 | 유형 | 핵심 개념 |
|----|------|------|-----------|
| G0-1 | 정면 입사 | 관찰 | θᵢ=0° → 꺾임 없음 |
| G0-2 | 굴절 조준 | 실시간 조준 | 공기→유리, 법선 쪽으로 꺾임 |
| G0-3 | 기울어진 보안창 | 레이저 회전 퍼즐 | 평행 유리판의 측면 이동 |
| G0-4 | 경보 구역 회피 | 정밀 계획·발사 | ±5° 큰 조준과 ±1° 미세 조준, 3발 제한 |
| G1-1 | 물의 굴절 | 예측·공개 | 물(n=1.33) vs 유리(n=1.50) 비교 |
| G1-2 | 다시 공기로 | 예측·공개 | 유리→공기, 법선에서 멀어짐 |
| G1-3 | 임계각 탐색 | 계획·발사 퍼즐 | 임계각 θc=arcsin(1/n)≈41.8° 체험 |
| G1-4 | 전반사 금고 | 계획·발사 퍼즐 | TIR 활용, 광섬유 원리 |
| G2-1 | 유리 너머의 반사 | 복합 계획·발사 퍼즐 | 유리 굴절 2회 → 거울 반사 |
| G2-2 | 순서가 잠금장치 | 순차 2발 퍼즐 | 거울→유리 키 획득 후 유리→거울 탈출 |

### 구현된 게임 장치

- 레이저 각도 조절 (±5°, 키보드 ←/→)과 G0-4 미세 조준 (±1°, Shift+←/→)
- 후반 작전형 스테이지의 계획 → 발사 → 결과 루프 (출사 경로 사전 비공개, 3발 제한)
- 스테이지 선택 지도 및 순차 해금
- 별점 3개 시스템 (조작 횟수 + 힌트 사용 여부)
- 추가 도전 과제 (힌트 없이 클리어, 최소 조작 등)
- 완료 스테이지 재입장 시 판만 초기화하고 최고 기록 보존
- 단계별 힌트와 고등학생 수준의 개념 중심 연구노트
- 물 블록(파란 점선 테두리), TIR 반사광(시안 맥동 광선)
- 소리 켬/끔, 소리 효과 (회전·성공 화음)
- 전반사 시각: 시안 빛 맥동 + ⟳ 전반사 링 표시
- 유리와 고정 거울의 교차점 순서를 계산하는 복합 광선 엔진
- 정답 선택 전 경로·센서를 숨기는 예측 미션과 첫 시도 별점
- 한국어/영어 전환

### 로드맵 vs. 구현 대조

| 로드맵 원안 | 구현 결과 | 비고 |
|---|---|---|
| 수직 입사 | G0-1 ✅ | |

| 다시 공기로 (유리→공기) | G1-2 ✅ | |
| 평행 유리판의 측면 이동 | G0-3, G0-4 ✅ | 공식 암기 대신 출사 위치 변화로 학습 |
| 굴절률 비교 | G1-1 ✅ | 물(n=1.33)과 유리(n=1.50) 예측 |
| 거울+유리 | G2-1, G2-2 ✅ | 장치 순서가 다른 복합 경로 |

### 검증 결과

- 공기→유리 30° 입사: 굴절각 19.47°
- 유리→공기 25° 입사: 굴절각 39.4°
- 유리→공기(n=1.5) 50° 입사: 전반사
- 임계각 arcsin(1/1.5) = 41.8° ✅ 확인
- G1-4 전반사 금고: 오른쪽 벽 TIR → 윗면 출사 → 좌상단 센서 도달 ✅
- G2-1 복합 작전: 유리 입·출사 → 고정 거울 반사 → 위쪽 센서 도달 ✅
- G2-2 순서 작전: A 거울→유리 암호 획득 → 유리→B 거울 잠금 해제 ✅
- G0-4 정밀 조준: −5° 큰 조준 뒤 −1° 두 번으로 −7° 센서 도달 ✅
- 브라우저 자동 광학 검증 27개 통과

### 완료 기준

- 광선이 임의 각도로 이동하고 셀 경계에서 끊기지 않는다.
- 스넬 법칙 계산과 전반사 판정이 테스트로 검증된다.
- 입사광·법선·굴절광이 색과 선 종류로 명확히 구분된다.
- 기존 반사 스테이지와 저장 데이터가 깨지지 않는다.

## Prototype 3. Focus Job — 렌즈와 초점

### 학습 목표

- 볼록 렌즈는 평행광을 초점으로 모으고 오목 렌즈는 퍼뜨린다.
- 렌즈 위치와 초점거리 변화가 상의 위치에 미치는 영향을 예측한다.
- 한 광선이 아니라 여러 대표 광선의 교점을 관찰한다.

### 새 게임 장치

- 볼록 렌즈, 오목 렌즈, 이동 가능한 스크린
- 초점 표식과 초점거리 조절 장치
- 한 점에서 출발하는 다중 광선 묶음

### 대표 스테이지

1. 평행광을 한 센서에 모으기
2. 초점거리가 다른 두 렌즈 비교
3. 스크린을 움직여 가장 선명한 위치 찾기
4. 오목 렌즈 뒤의 가상 초점 추적
5. 거울과 렌즈를 결합한 망원 경로
6~8. 미끼 렌즈와 제한 이동 횟수를 사용한 응용 퍼즐

### 양자정보 연결

단일 광자도 렌즈로 집광해 검출기에 보낼 수 있다는 점을 Photon
Operations에서 다시 사용합니다.

## Prototype 4. Spectrum Vault — 프리즘과 색 분리

### 학습 목표

- 백색광이 여러 파장의 빛으로 구성됨을 관찰한다.
- 유리의 굴절률이 파장에 따라 달라 색이 분리됨을 이해한다.
- 색 혼합과 색 분리를 서로 다른 조작으로 구분한다.

### 새 게임 장치

- 회전 프리즘, 백색광 광원, RGB 광선
- 빨강·초록·파랑 센서와 백색광 센서
- 색 필터와 파장별 굴절률 표시

### 대표 스테이지

1. 백색광을 프리즘에 통과시키기
2. 빨강과 파랑 중 더 크게 꺾이는 색 예측
3. 세 색을 각각 맞는 센서로 보내기
4. 필터로 특정 색만 통과시키기
5. 두 프리즘으로 색을 다시 합치기
6~8. 거울·렌즈·프리즘 복합 금고

### 엔진 요구사항

한 광선이 여러 파장 광선으로 분기되는 다중 광선 추적이 필요합니다.

## Prototype 5. Polarization Cipher — 편광과 측정 기저

### 학습 목표

- 빛의 진행 방향과 편광 방향이 다른 정보임을 구분한다.
- 0°, 45°, 90° 편광 필터를 회전해 투과 세기를 예측한다.
- 서로 수직인 필터 사이에 45° 필터를 넣으면 일부 빛이 통과함을 체험한다.
- 편광 필터가 이후 양자 챕터의 측정 기저가 됨을 준비한다.

### 새 게임 장치

- 편광 광원, 회전 편광판, 분석기, 세기 센서
- 편광 방향 화살표와 밝기 게이지
- 원형 편광은 후반 보너스 스테이지에서만 도입

### 8개 스테이지 흐름

1. 같은 방향 필터 통과
2. 수직 필터에서 차단
3. 45° 필터의 중간 밝기
4. 세 장의 필터 역설
5. 목표 밝기 맞추기
6. 경로와 편광을 동시에 맞추기
7. 미지 편광 상태 판별
8. 편광 암호 금고

### 수학 연결

말뤼스 법칙 `I = I₀ cos² θ`는 밝기 실험 뒤에 연구노트로 도입합니다.

## Prototype 6. Wave Protocol — 간섭과 회절

### 학습 목표

- 위상과 경로 길이 차이가 밝고 어두운 결과를 만든다.
- 보강 간섭과 상쇄 간섭을 예측한다.
- 슬릿 폭과 파장에 따라 회절 무늬가 달라짐을 비교한다.
- 간섭과 회절이 함께 나타나지만 같은 말은 아님을 구분한다.

### 새 게임 장치

- 빔 스플리터, 위상 지연판, 경로 길이 조절기
- 결합기, 간섭 센서, 단일·이중 슬릿, 스크린
- 진폭과 위상을 가진 광선 상태

### 10개 스테이지 흐름

1~3. 같은 위상과 반대 위상 비교
4~6. 마하–젠더 간섭계의 두 경로 조절
7. 한쪽 경로를 막았을 때 변화
8. 단일 슬릿 회절
9. 이중 슬릿 간섭 무늬
10. 파장·슬릿·경로를 함께 맞추는 파동 금고

### 양자정보 연결

동일한 마하–젠더 장치를 Photon Operations에서 단일 광자 모드로 다시
사용하여 “한 번의 검출”과 “반복했을 때의 간섭 무늬”를 연결합니다.

## Prototype 7. Photon Operations — 광자와 확률

### 학습 목표

- 빛의 세기와 한 번의 광자 검출을 구분한다.
- 개별 결과는 확률적이지만 반복 측정에는 안정된 분포가 나타남을 본다.
- 측정 장치와 기저 선택이 결과 확률을 바꿈을 체험한다.
- 고전 파동 설명과 광자 검출 설명이 각각 무엇을 말하는지 비교한다.

### 새 게임 장치

- 단일 광자 광원, 클릭 검출기, 반복 발사 버튼
- 검출 횟수·히스토그램·예측 확률 표시
- 시드 기반 난수로 재현 가능한 학습 실험

### 대표 스테이지

1. 광자 한 개 발사와 검출 클릭
2. 10회·100회 반복 결과 비교
3. 50:50 빔 스플리터의 확률
4. 편광 상태와 측정 기저
5. 경로 정보가 있을 때 간섭 변화
6~9. 확률 목표를 맞추는 실험 설계
10. 양자정보 챕터 입장 시험

---

## Prototype 8. Quantum Heist — 양자정보

### 학습 목표

- 편광을 `|H⟩`, `|V⟩` 큐비트로 표현한다.
- 중첩과 측정 결과를 상태 자체와 구분한다.
- 서로 다른 측정 기저가 호환되지 않음을 실험으로 확인한다.
- 복제 불가능성과 측정 교란을 BB84 도청 탐지에 연결한다.

### 새 게임 장치

- 상태 준비기, H/V 및 ± 측정기, 단일 큐비트 게이트
- 블로흐 구면은 조작 경험을 쌓은 뒤 시각화 도구로 제공
- Alice·Bob·Eve 통신 보드와 오류율 표시

### 10개 스테이지 흐름

1~3. 상태 준비와 한 기저 측정
4. 다른 기저에서 나타나는 확률
5. 연속 측정과 상태 변화

## 공통 난이도 구조

난이도 선택으로 같은 스테이지를 세 벌 만들지 않습니다.

- 기본 캠페인: 핵심 개념을 순서대로 학습
- 힌트: 막힌 플레이어를 위한 단계별 지원
- 별 3개: 적은 조작과 힌트 미사용 목표
- 추가 도전: 미끼 무시, 제한 검출 횟수, 목표 확률 등 전문가 목표
- 후반 스테이지: 이전 챕터 장치를 섞되 새 개념은 한 번에 하나만 추가

## 엔진 개발 순서

### Milestone A — 연속 각도 광선 엔진 (핵심 완료)

- 위치와 방향을 실수 벡터로 저장

- 스넬 법칙과 전반사 판정
- SVG 경로를 임의 각도 선분으로 렌더링
- 기존 4방향 거울 캠페인은 안정성을 위해 격자 전용 `raycaster.js`를 유지하며, 공통 엔진 통합은 보류

### Milestone B — 다중 광선과 광학 상태

Prototype 3~6에서 확장합니다.

- 여러 대표 광선과 분기 광선

- 렌즈·프리즘·빔 스플리터 장치 인터페이스

### Milestone C — 확률적 측정 엔진

Prototype 7~8에서 확장합니다.

- 단발 측정과 반복 실험
- 시드 기반 확률 샘플링과 히스토그램
- 큐비트 상태와 측정 기저
- 고전 광선 모드와 광자 모드의 명확한 UI 분리

## 제작 원칙

1. 새 챕터는 한 가지 새 물리 규칙만 먼저 소개합니다.
2. 수식은 조작 결과를 예측할 필요가 생긴 뒤 보여줍니다.
   게임 화면에서는 개념을 먼저 설명하고, 교육과정 밖의 유도식은 심화 자료로 분리합니다.
3. 실제 물리와 어긋나는 단순화는 게임 안에서 명시합니다.
4. 모바일에서도 각도·색·밝기 정보를 색상 하나에만 의존하지 않습니다.
5. 각 챕터의 장치는 이후 양자정보 챕터에서 다시 사용합니다.
6. 물리 계산은 시각 효과와 분리해 자동 테스트할 수 있게 유지합니다.
